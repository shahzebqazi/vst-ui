Executive design requirements

White paper · VST UI mockups · Living design system

Purpose

This document states the executive-level design contract for the VST UI mockups experience: a static, GitHub Pages–hosted shell for exploring screen mockups, themes, and design documentation in one place.

Product surface

  • Two-column layout: controls (themes, mockups, documents) and a fixed mobile viewport (390×844).
  • CSS-only themes (Blueprint, Paper, Dark Dots) for quick visual context.
  • Document links open full-page artifacts (this white paper and future design docs).

Audience & outcomes

  • Design / PM: align on IA, labels, and screen inventory without a build step.
  • Stakeholders: skim requirements and visual direction via mockups and written specs.
  • Engineering: use mockups and docs/ as the single source of truth for scope.

Non-goals (current phase)

  • No runtime plugin or DAW integration in this repository.
  • No authentication or dynamic CMS; content is versioned in Git.

Success criteria

  • Anyone with the URL can open mockups and key design documents within two clicks.
  • New documents can be added as buttons under Documents without changing layout rules.
  • Requirements and menu inventory in docs/ stay in sync with the published shell.

Canonical technical requirements: docs/REQUIREMENTS.md (repository).
